# Methodz Meeting Manager v0.8 Notes

## Release Theme

Version 0.8 is the **history, recovery, and accessibility build**.

The app remains static, offline-first, dependency-free, and deployable by opening `meeting.html` directly or publishing the repository to any static host.

## Added

### Record Revision History

Every successful meeting save now creates a revision snapshot under the browser-local `methodzMeetingRevisions` key.

Revision history includes:

- revision number
- capture time
- save or restore reason
- content hash
- complete meeting-record snapshot

Saved-record cards now include **Revision History**. A historical revision can be previewed or restored. Before a revision restore, the current record is automatically preserved as another revision.

The default retention limit is 50 revisions per record and can be changed through `revisionLimit` in `config.js`.

### Non-Destructive Archive Vault

The former saved-record Delete action is now an **Archive** action.

Archiving:

1. copies the complete record into `methodzArchivedMeetingRecords`
2. removes the record from the active record collection
3. keeps its revision history intact
4. makes it available in the Archive Vault

The Archive Vault supports:

- restore to active records
- archived-record JSON download
- permanent deletion with explicit confirmation

If a restored record ID conflicts with an active record, the restored copy receives a new ID and keeps a `restoredFromRecordId` reference.

### Workspace Backup and Restore

The Workspace Backup panel exports one package containing all recognized Methodz Meeting Manager storage entries, including:

- active records
- archived records
- revision history
- draft
- attendee directory
- organization directory
- templates
- numbering settings
- organization presets
- sync metadata
- future `methodz*` local-storage entries

Workspace packages include a package type, package version, schema version, timestamp, summary, entries, and checksum.

Before applying a restore package, the app stores a complete pre-restore recovery package under `methodzPreRestoreBackup`. That recovery package can be downloaded from the same panel.

### Adapter Contract Tests

`adapter-contract-tests.js` adds an in-app isolated test harness.

The harness verifies:

- adapter manager availability
- required contract methods
- empty-list behavior
- create and update through `upsertRecord`
- `getRecord`
- collection replacement
- targeted deletion
- export-envelope metadata
- health checking

Tests run against a temporary local-storage key and remove it afterward. Active meeting records are not modified.

### Accessibility and Keyboard Navigation

Version 0.8 adds:

- skip-to-workspace link
- live status announcements
- explicit main landmark focus target
- generated labels and IDs for dynamic form fields
- stronger visible focus indicators
- reduced-motion support
- keyboard shortcut guide

Keyboard shortcuts:

| Shortcut | Action |
|---|---|
| Ctrl / Command + S | Save current meeting |
| Ctrl / Command + Shift + N | Start a new meeting |
| Alt + R | Focus saved-record search |
| Alt + A | Jump to Archive Vault |
| Alt + H | Toggle keyboard shortcut guide |
| Escape | Close shortcut guide or revision history |

## Storage Keys Added

```text
methodzMeetingRevisions
methodzArchivedMeetingRecords
methodzPreRestoreBackup
methodzAccessibilityPreferences
```

The configuration schema and adapter contract version are now `0.8.0`.

## Compatibility

- Existing v0.7 and earlier records remain readable.
- No automatic destructive migration is performed.
- Existing records gain revision history only after their next successful save.
- The status value `Archived` remains available for business workflow use, while the Archive Vault is a separate storage lifecycle action.

## Deployment

No build step was added.

Keep these v0.8 files beside the existing app files:

```text
features-v08-history.js
features-v08-workspace.js
adapter-contract-tests.js
features-v08-accessibility.js
features-v08.css
```

Then deploy the repository directory normally or open `meeting.html` directly.
