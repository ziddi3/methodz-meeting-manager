# Methodz Meeting Manager v1.6.2 Release Notes

## Purpose

Version 1.6.2 adds a public-key custody and operational evidence layer on top of the v1.6 ECDSA package-signing system. The release does not change the meeting-record schema, require a server, or persist private signing keys.

## Added

- Portable `key-custody-core.js` for public custody-manifest construction and validation.
- Browser workspace for recording rotation, revocation, lost-key response, and recovery-rehearsal events.
- Public custody-manifest export containing public JWKs, key IDs, lifecycle state, and custody events.
- Separate custody-audit export.
- Completed-event controls requiring an operator, witness, effective date, reason, and four custody checklist confirmations.
- Fail-closed validation for mismatched key IDs, same-key rotations, unknown key references, invalid dates, incomplete completed events, and private JWK material.
- Node and Playwright tests for core validation, workspace loading, completed rotation ceremonies, and incomplete-checklist rejection.
- App-shell cache version 1.6.2 with all new static assets.

## Security boundaries

- Private JWK material is prohibited from custody storage and custody exports.
- A custody event does not automatically change public-key registry status.
- Public-key fingerprints must be confirmed through a separate trusted channel.
- Browser-local custody records are operational evidence, not immutable proof of identity, authority, approval, or delivery.
- Signing and verification remain optional. Core meeting workflows still work through direct-file operation.

## Compatibility

- Meeting-record schema remains `1.6.0`.
- App-shell version becomes `1.6.2`.
- Existing v1.6 signed packages and public-key registry entries remain compatible.
- No migration is required because the new custody event collection is additive and independently keyed.
