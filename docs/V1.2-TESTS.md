# Methodz Meeting Manager v1.2 Test Checklist

## Automated checks

GitHub Actions should pass:

- JavaScript syntax checks for every `.js` file;
- required v1.2 file checks;
- HTML and service-worker wiring checks;
- existing v1.0 and v1.1 regression tests;
- `tests/v12-smoke.spec.js`.

## Manual external-export approval

1. Open `meeting.html`.
2. Enter a title and verify the date.
3. Choose Partner Safe and Approved Partner Organization.
4. Request review with a requester name and purpose note.
5. Confirm download remains blocked before approval.
6. Approve as Administrator or Auditor with a reviewer name.
7. Preview the external package.
8. Confirm the package contains approval metadata and a recomputed integrity digest.
9. Download JSON and HTML.
10. Export the approval log and confirm approved-download events exist.
11. Change the meeting title or content.
12. Confirm the prior approval becomes invalid.

## Recipient-policy checks

- Approved Partner Organization permits Partner Safe only.
- Public Communications permits Public Summary only.
- Contractual / Custom Recipient permits Partner Safe or Custom.
- Custom requires a purpose and sharing-limits note.

## Role checks

Switch workspace role and verify only configured approval roles can:

- approve external exports;
- reject or revoke external approvals;
- approve disposition;
- reject or revoke disposition.

## Disposition approval

1. Save and archive a meeting without an active hold.
2. Attempt permanent deletion before approval and confirm it is blocked.
3. Request disposition review with a requester and basis.
4. Approve with an authorized reviewer.
5. Confirm the panel reports approval for the current fingerprint.
6. Permanently delete through the Archive Vault confirmation.
7. Export the disposition audit and confirm the approval is consumed and a completion event exists.

## Preservation hold precedence

1. Save a record with an active preservation hold.
2. Archive it.
3. Confirm permanent deletion remains disabled or blocked.
4. Confirm disposition approval cannot override the hold.
5. Release the hold through the controlled record, save, and re-evaluate.

## Direct-file compatibility

Open `meeting.html` directly from the filesystem and confirm:

- core meeting workflows still function;
- approval panels load;
- browser-local approval registries persist;
- service-worker registration is skipped as expected;
- external-package integrity uses the clearly labeled compatibility fallback when Web Crypto is unavailable.
