# Methodz Meeting Manager v1.2 Test Checklist

## Load and migration

- Open `meeting.html` from a static server.
- Confirm External Export Approval appears below Partner-Safe Export.
- Confirm the configuration schema reports `1.2.0`.
- Import or open a pre-1.2 record and confirm `externalReleaseControl` is added without losing unknown fields.

## Destination policies

- Select Public / Website with Partner Safe and confirm approval request is blocked.
- Select Public / Website with Public Summary and confirm approval request succeeds.
- Select Canadian Soft Water Corporation with Custom External Copy and Discussion Notes enabled; confirm the request is blocked.
- Disable Discussion Notes and confirm the conservative custom allow-list succeeds.

## Approval lifecycle

- Create a request with requester, purpose, and expiry.
- Confirm status is Pending.
- Enter reviewer information and approve it.
- Confirm status is Approved and the fingerprint is displayed.
- Reject a separate pending request.
- Revoke an approved request.
- Confirm rejected, revoked, and expired approvals cannot release packages.

## Fingerprint protection

- Approve a package.
- Change the meeting title, summary, decision, task, destination, profile, or allowed custom section.
- Confirm download is blocked until a new approval is created and approved.
- Restore the exact approved content and confirm the matching approval is recognized while unexpired.

## Approved downloads

- Download approved JSON.
- Confirm `packageType` is `methodz-approved-external-meeting-copy`.
- Confirm approval metadata contains requester, reviewer, destination, purpose, approval ID, and fingerprint.
- Confirm typed signatures and signature verification metadata are absent.
- Confirm package integrity is recalculated after approval metadata is attached.
- Repeat for HTML and inspect the release-approval notice.

## Audit and record metadata

- Export the approval audit package.
- Confirm request, approve/reject/revoke, and download events are present.
- Release an approved copy from an active saved record.
- Confirm `externalReleaseControl.lastApprovalId`, `lastApprovedAt`, and `lastExportAt` are updated.

## Offline and regression

- Repeat the workflow in direct-file mode.
- Confirm core save, archive, revision, retention, legal-hold, redaction, import, backup, and print workflows remain available.
- Run GitHub Actions static and Playwright checks.
