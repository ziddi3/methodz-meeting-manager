# Methodz Meeting Manager v1.4 Test Checklist

## Static loading

1. Open `meeting.html` directly from the file system.
2. Confirm the page loads without a blank screen or JavaScript error.
3. Confirm **Recipient-Specific Export Policies** appears after **External Export Approval**.
4. Confirm the displayed schema through the migration tools is `1.4.0`.
5. Open `archive.html` and confirm existing records still render.

## Create a recipient policy

1. Enter a policy label.
2. Enter a named recipient or accountable contact.
3. Enter an organization and optional contact reference.
4. Choose a base destination policy.
5. Leave Partner Safe and Custom External Copy enabled.
6. Leave the default field allow-list selected.
7. Set a future review date.
8. Save the policy.
9. Confirm it appears in the saved policy selector and policy cards.
10. Refresh the browser and confirm the policy remains available.

## Apply and preview

1. Select the policy.
2. Choose **Apply to Export**.
3. Confirm the External Export destination changes to a value beginning with `recipient:`.
4. Enter meeting information, a summary, and at least one Follow-Up Task.
5. Remove the Tasks field group from the recipient policy and save it.
6. Reapply the policy.
7. Preview the external copy.
8. Confirm the package contains the summary but no `tasks` field.
9. Confirm `manifest.recipientPolicy` contains the selected policy snapshot.
10. Confirm package integrity is present and the status names the recipient policy.

## Profile and custom-section enforcement

1. Create a policy that allows only Partner Safe.
2. Apply it and confirm the profile changes to Partner Safe when necessary.
3. Create a policy allowing Custom External Copy but not attachments.
4. Select Custom External Copy and enable attachments before applying the policy.
5. Apply the policy and confirm the attachment checkbox is cleared.
6. Attempt to re-enable attachments and request approval.
7. Confirm the existing destination-policy validation blocks the request.

## Sensitive discussion notes

1. Start a new recipient policy.
2. Enable **Discussion notes (sensitive)**.
3. Leave the verification note blank.
4. Attempt to save.
5. Confirm saving is blocked.
6. Enter a meaningful verification note describing the recipient and permitted use.
7. Save again and confirm success.
8. Confirm the selected redaction profile and destination policy still control whether notes can be included.

## Approval binding

1. Apply a valid recipient policy.
2. Preview the external copy.
3. Enter Requested By, Release Purpose, and Approval Expires.
4. Request approval.
5. Export or inspect browser storage.
6. Confirm the approval has:
   - a `destinationPolicyId` beginning with `recipient:`;
   - `recipientPolicyId`;
   - `recipientPolicySnapshot`;
   - a content fingerprint.
7. Approve the request with a separate reviewer.
8. Download the approved JSON package.
9. Confirm its redaction manifest contains the recipient-policy snapshot.

## Approval invalidation after policy change

1. Create and approve a package where Tasks are allowed and contain data.
2. Edit the recipient policy and remove Tasks.
3. Reapply the policy.
4. Attempt to download using the previous approval.
5. Confirm download is blocked because the redacted content fingerprint changed.
6. Create and approve a new request for the narrower package.

## Review date and status

1. Set a recipient policy review date in the past.
2. Attempt to apply it.
3. Confirm the app blocks use and reports that the policy is past review.
4. Set a future review date and save.
5. Deactivate the policy.
6. Confirm its dynamic destination is removed and it cannot be applied.
7. Reactivate it and confirm it returns.

## Import, export, and audit

1. Export recipient policies JSON.
2. Export the recipient-policy audit JSON.
3. Open both files and confirm they are valid JSON.
4. Import the policy package into a clean browser profile.
5. Confirm policies merge by ID and prefer the newest `updatedAt` value.
6. Confirm import does not alter meeting records.

## Regression

1. Save, edit, archive, restore, and revise a meeting.
2. Run revision comparison.
3. Create a complete workspace backup.
4. Preview Partner Safe and Public Summary exports without a recipient-specific policy.
5. Confirm generic destination policies still function.
6. Confirm preservation holds still block permanent disposition.
7. Confirm disposition approval and event-chain verification still function.
8. Confirm direct-file mode requires no server, package installation, or build command.
