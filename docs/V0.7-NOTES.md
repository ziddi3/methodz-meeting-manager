# Version 0.7 Notes

## Release Theme

Version 0.7 adds a dedicated archive page, a real record-storage adapter boundary, stronger print output, and a reusable organization / representative directory.

The app remains static, dependency-free, offline-first, and deployable without a build step.

## Added

### Dedicated Archive Detail Page

`archive.html` and `archive.js` provide a focused record view outside the editing workspace.

The archive page includes:

- meeting information
- organization snapshots
- attendance and typed signatures
- signature audit
- agenda status
- free-form notes
- structured decisions
- follow-up tasks
- attachment references
- summary
- record timestamps and schema audit

Saved record cards now receive an **Archive Page** action. The main form also receives an **Open Archive Preview** action for the current unsaved meeting.

### Edit Handoff

A saved record can be opened on `archive.html`, then returned to `meeting.html` through **Edit Record**.

The handoff uses `sessionStorage` only to carry the selected record ID between pages. The saved record remains in the active data adapter.

### Data Adapter Contract

`data-adapter.js` introduces `window.MethodzMeetingData`.

The default `LocalStorageMeetingAdapter` implements:

- `listRecords()`
- `getRecord(recordId)`
- `replaceRecords(records)`
- `upsertRecord(record)`
- `deleteRecord(recordId)`
- `createExportEnvelope(extra)`
- `healthCheck()`

Version 0.7 routes the core global `getRecords()` and `setRecords()` functions through the adapter.

This does not enable cloud sync. It creates the interface required for a future cloud provider.

### Organization / Representative Directory

`features-v07-organizations.js` adds a browser-local directory with:

- name
- type
- primary representative
- contact detail
- notes
- JSON export/import
- selection into the current meeting
- optional representative insertion into attendance

Selected directory entries are copied into the meeting record as `organizationDetails` snapshots.

### Print Improvements

`features-v07.css` adds print rules for:

- archive record cards
- attendance tables
- signature audit sections
- tasks
- attachment references
- structured decisions
- record audit metadata

Interactive dashboards and management controls are hidden during print.

## Storage

New key:

```text
methodzOrganizationDirectory
```

Existing meeting records remain under:

```text
methodzMeetingRecords
```

## Compatibility

Older records without `organizationDetails` remain readable.

The archive page handles missing optional sections by showing a clear empty-state message instead of failing.

## Known Limits

- Cloud data is not transmitted.
- Attachment References remain metadata only.
- `archive.html` needs access to the same browser storage origin as `meeting.html`.
- Some browsers handle `file://` local storage differently. A small local static server is the most consistent testing environment.
