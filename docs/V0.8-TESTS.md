# Methodz Meeting Manager v0.8 Manual Tests

Run these checks in a browser using `meeting.html`.

## 1. Startup and Compatibility

- [ ] Open `meeting.html` directly from disk.
- [ ] Confirm the page loads without a build command or server.
- [ ] Confirm existing saved records still appear.
- [ ] Confirm the Archive Vault, Workspace Backup, Adapter Contract Tests, and Keyboard Help controls appear.
- [ ] Open the browser console and confirm there are no startup errors.

## 2. Revision History

- [ ] Create a meeting with a title and date.
- [ ] Save the meeting.
- [ ] Confirm the saved-record card includes **Revision History**.
- [ ] Open Revision History and confirm Revision 1 exists.
- [ ] Change notes, tasks, or summary and save again.
- [ ] Confirm another revision appears.
- [ ] Preview a revision as JSON.
- [ ] Restore an older revision.
- [ ] Confirm the form reloads with the older values.
- [ ] Confirm the pre-restore current state and restored state are both represented in history.
- [ ] Reload the page and confirm revision history persists.

## 3. Archive Vault

- [ ] Select **Archive** on a saved record.
- [ ] Cancel the first confirmation and confirm nothing changes.
- [ ] Archive the record after confirming.
- [ ] Confirm the record disappears from active Saved Meeting Records.
- [ ] Confirm the record appears in Archive Vault.
- [ ] Download the archived record and inspect the JSON.
- [ ] Restore the archived record.
- [ ] Confirm it returns to active Saved Meeting Records.
- [ ] Archive it again.
- [ ] Test Permanent Delete and cancel.
- [ ] Confirm the archived record remains.
- [ ] Permanently delete only a disposable test record.
- [ ] Confirm it is removed from Archive Vault.

## 4. Workspace Backup

- [ ] Add or update at least one meeting, attendee directory entry, organization directory entry, and custom template.
- [ ] Export a Workspace Backup.
- [ ] Confirm the file name begins with `methodz-workspace-backup-`.
- [ ] Open the JSON and confirm `packageType` is `methodz-meeting-manager-workspace`.
- [ ] Confirm `checksum`, `entries`, and `summary` are present.
- [ ] Confirm active records, revisions, and archive data are included when available.

## 5. Workspace Restore

Use a disposable browser profile or export a backup before testing.

- [ ] Choose a valid workspace package.
- [ ] Confirm a restore preview appears before any data changes.
- [ ] Confirm the preview reports active records, archived records, revision groups, and storage entries.
- [ ] Cancel the restore and confirm current data remains unchanged.
- [ ] Choose a non-Methodz JSON file and confirm it is rejected.
- [ ] Modify a valid package without updating its checksum and confirm it is rejected.
- [ ] Apply a valid restore package.
- [ ] Confirm the application reloads.
- [ ] Confirm restored records and directories appear.
- [ ] Confirm **Download Pre-Restore Recovery** becomes available.
- [ ] Download the recovery package and verify it represents the workspace from before restore.

## 6. Adapter Contract Tests

- [ ] Run Adapter Contract Tests.
- [ ] Confirm all tests report PASS.
- [ ] Confirm the badge reports the passed total.
- [ ] Download the test report.
- [ ] Confirm the report contains contract version, active adapter, totals, and individual results.
- [ ] Confirm no temporary `methodzAdapterContractTest:*` key remains in local storage.
- [ ] Confirm active meeting records were not modified.

## 7. Keyboard Navigation

- [ ] Press Tab at page start and confirm the skip link becomes visible.
- [ ] Activate the skip link and confirm focus moves to the main workspace.
- [ ] Press Alt + H and confirm Keyboard Help opens.
- [ ] Press Escape and confirm Keyboard Help closes.
- [ ] Press Alt + R and confirm focus moves to saved-record search.
- [ ] Press Alt + A and confirm focus moves to Archive Vault.
- [ ] Enter a valid meeting title/date and press Ctrl/Command + S.
- [ ] Confirm the browser save-page dialog does not open and the meeting record saves instead.
- [ ] Press Ctrl/Command + Shift + N and confirm the new-meeting workflow runs.

## 8. Dynamic Field Accessibility

- [ ] Add an attendee and confirm focus moves to its first field.
- [ ] Add a task and confirm focus moves to its first field.
- [ ] Add a structured decision and confirm focus moves to its first field.
- [ ] Add an attachment reference and confirm focus moves to its first field.
- [ ] Inspect dynamic inputs and confirm each has an ID and associated label.
- [ ] Confirm signature fields reference the hidden signature explanation.
- [ ] Navigate all controls using only Tab and Shift + Tab.
- [ ] Confirm focus is always visibly identifiable.

## 9. Reduced Motion and Mobile

- [ ] Enable reduced motion in the operating system or browser.
- [ ] Confirm jumps to records, history, and archive do not use prolonged smooth animation.
- [ ] Test at a narrow mobile width.
- [ ] Confirm archive and revision cards collapse to one column.
- [ ] Confirm buttons remain readable and touch-friendly.

## 10. Regression

- [ ] Draft auto-save and restore still work.
- [ ] Templates still apply.
- [ ] Attendee directory still works.
- [ ] Organization directory still works.
- [ ] Structured decisions still save and reload.
- [ ] Attachment references still save and reload.
- [ ] Meeting archive page still opens.
- [ ] TXT, JSON, HTML, CSV, and print exports still work.
- [ ] Record search and filters still work.
- [ ] Duplicate review and sync package export still work.

## Release Gate

Version 0.8 is ready for wider use when:

- all JavaScript files pass syntax checks
- all adapter contract tests pass
- no existing v0.7 workflow regresses
- archive restore is verified with a disposable record
- workspace restore is verified in a disposable browser profile
- keyboard-only navigation is usable from top to bottom
