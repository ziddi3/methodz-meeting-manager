# Methodz Meeting Manager v1.3 Changelog

## Added

- `config-v13.js` with disposition approval roles, storage keys, separation-of-duty settings, and local registry limits.
- `migrations-v13.js` with `dispositionControl` defaults and schema validation.
- `features-v13-disposition.js` with disposition requests, approval, rejection, revocation, consumption, fingerprint validation, and Archive Vault gating.
- Browser-local preservation and disposition event chain with verification and JSON export.
- Legal-hold placement and release transition events.
- Archive Vault badges for preservation holds, pending review, approval required, and approved disposition.
- `archive-v13.js` with read-only disposition and preservation audit details.
- `features-v13.css` and `tests/v13-smoke.spec.js`.
- v1.3 release, architecture, and test documentation.

## Changed

- Bumped the active schema and app-shell version to `1.3.0`.
- Permanent Archive Vault removal now requires a matching approved disposition request in addition to the existing final confirmation.
- Active preservation holds remain the stronger blocking rule.
- The service-worker cache now includes v1.3 assets.
- v1.0 through v1.2 regression suites now run under schema 1.3.0.
- GitHub Actions validates v1.3 wiring and browser behavior.

## Safety notes

- Browser-local requester, reviewer, role, fingerprint, and audit data are workflow safeguards, not authenticated authorization.
- The local event chain can detect some changes but is not immutable or remotely anchored.
- The FNV-1a-32 record fingerprint and chain digest are not digital signatures.
- A future remote provider must enforce holds, roles, separation of duties, record versions, approval consumption, and append-only audit storage on the server.
