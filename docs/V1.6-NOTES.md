# Methodz Meeting Manager v1.6 Notes

## Release focus

Version 1.6 adds optional public-key signatures for exported JSON packages while preserving the existing recipient-policy operations, release receipts, static deployment, and offline-first record model.

## Added

- ECDSA P-256 signatures with SHA-256 through Web Crypto.
- Explicit signing-key generation and JWK import/export.
- Memory-only private-key handling.
- Browser-local public-key registry with Active and Revoked states.
- Signed-package verification and verification-report export.
- Standalone `verify.html` entry point.
- Package-signing and verification audit export.
- Record metadata for the most recent package-signing operation.
- Public-key registry sanitation and private-material rejection.
- Automated package-tampering, signature-metadata-tampering, and private-key-leakage coverage.

## Private-key rule

The application never writes private JWK material to localStorage. A generated or imported private key exists only in the current page memory. The operator must explicitly download and protect a private-key backup before refreshing or closing the page.

The cryptographic core blocks attempts to sign packages containing private JWK material. Public-key imports are normalized before storage or export, and malformed registry entries are sanitized when the workspace loads.

## What a valid signature means

A valid signature confirms that the JSON package content and displayed signature metadata match the bytes signed by the holder of the corresponding private key.

It does not independently prove:

- the legal identity of the signer;
- the authority of the signer;
- recipient identity;
- package delivery;
- approval legitimacy;
- compliance with a legal electronic-signature regime.

Verify the public key ID through an independent trusted channel before relying on the signer label.

## Deployment

No runtime dependency, framework, server, or build step was added. Core meeting workflows still operate in direct-file mode. Cryptographic operations require Web Crypto, which is normally available on HTTPS or localhost and may vary in direct-file contexts.
