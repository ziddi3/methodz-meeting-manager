# Methodz Meeting Manager v1.3 Test Checklist

## Automated checks

GitHub Actions should pass:

- JavaScript syntax checks;
- required v1.3 file checks;
- HTML and service-worker wiring checks;
- v1.0, v1.1, and v1.2 regression suites under schema 1.3.0;
- `tests/v13-smoke.spec.js`.

## Manual disposition review

1. Save and archive a meeting without an active preservation hold.
2. Attempt Permanent Delete before approval.
3. Confirm the action is blocked and the disposition panel receives focus.
4. Enter Requested By and a documented Disposition Basis.
5. Request disposition review.
6. Enter a different Reviewed By name.
7. Approve as Administrator or Auditor.
8. Confirm the Archive Vault badge reads Disposition Approved.
9. Accept the existing final confirmation.
10. Confirm the archive entry is removed and the approval becomes Consumed.

## Separation of duties

1. Create a new disposition request.
2. Enter the same normalized name for Requested By and Reviewed By.
3. Confirm approval is blocked.
4. Confirm the request remains Pending.

## Role checks

Switch workspace role and verify:

- Administrator can approve, reject, and revoke;
- Auditor can approve, reject, and revoke;
- other local roles are blocked from review actions.

These are workflow checks only, not authentication tests.

## Fingerprint invalidation

1. Create a disposition request for an archived record.
2. Change the archived record JSON through a controlled test fixture or development console.
3. Confirm approval fails because the fingerprint changed.
4. Create a new request for the changed record.

## Preservation-hold precedence

1. Save a meeting with an active preservation hold.
2. Archive it.
3. Confirm the Archive Vault removal control remains disabled.
4. Confirm a disposition request is blocked.
5. Confirm no approval can override the hold.

## Event-chain checks

1. Place a preservation hold and save the record.
2. Save again without changing the hold.
3. Confirm only one `legal-hold-placed` event exists.
4. Release the hold and save.
5. Confirm one `legal-hold-released` event exists.
6. Request, approve, and complete disposition for another record.
7. Run Verify Event Chain.
8. Confirm the chain reports valid.
9. Modify one stored event in a development-only test.
10. Confirm verification reports a digest or link mismatch.

## Audit exports

Export both:

- Disposition Approval Audit;
- Preservation Event Chain.

Confirm each package contains a clear notice that browser-local history is not immutable or authenticated.

## Archive page

Open a record through its Archive Page button and confirm the read-only Disposition & Preservation Audit section displays associated approvals and record events.

## Direct-file compatibility

Open `meeting.html` directly and verify the v1.3 panels and local workflow still function without service-worker support or a build command.
