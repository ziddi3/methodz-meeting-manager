# v1.6 Change Log

## Added

- Optional ECDSA P-256 / SHA-256 signatures for JSON packages.
- Canonical package and signature-metadata binding.
- Embedded public verification key and derived key ID.
- Private and public JWK import and export.
- Memory-only private-key workspace.
- Public-key registry with local Active and Revoked states.
- Public-key registry sanitation and private-material rejection.
- Signing and verification audit exports.
- Standalone signed-package verifier.
- Schema 1.6 migration and `externalSignatureControl` record metadata.
- Node Web Crypto and Playwright cryptographic regression coverage.

## Changed

- Active schema version: `1.6.0`.

- App manifest includes a standalone verifier shortcut.

## Preserved

- Static deployment with no runtime framework or build step.
- Direct-file core meeting workflows.
- v1.5 recipient-policy governance and release receipts.
- Destination-bound export approval.
- Redaction and recipient allow-list enforcement.
- Retention, preservation holds, and disposition approval.
- Archive, revision, recovery, workspace backup, directory, template, task, and consent workflows.

## Trust boundary

Package signatures add cryptographic integrity relative to a key. They do not authenticate people, roles, approvals, recipients, delivery, or legal compliance. Private-key custody and trusted public-key distribution remain operational responsibilities outside the browser-local application.
