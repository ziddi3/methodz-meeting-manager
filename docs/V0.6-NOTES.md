# Version 0.6 Notes

Version 0.6 adds control and governance around the existing offline meeting workflow.

The app remains static, offline-first, dependency-free, and openable directly from `meeting.html`.

## Added

- Meeting Numbering Settings
- Organization Presets
- Duplicate Record Review
- Duplicate report export
- Local Sync Readiness
- Local sync queue metadata
- Sync package JSON export

## Meeting Numbering

The default numbering style remains simple:

```text
001
002
003
```

Optional settings can add:

- a prefix
- the current year
- a custom minimum digit length
- a manually chosen next sequence number

Changing numbering affects new records only. Existing saved records keep their original numbers.

## Organization Presets

Default presets are defined in `config.js`.

Custom presets are stored locally in the browser and can be exported/imported as JSON.

This is meant to make common representative combinations fast without changing the base organization list.

## Duplicate Record Review

Duplicate Review scans saved records for likely duplicates using:

- same normalized title and date
- same date and attendee list

It is advisory only. It does not merge, delete, or change records automatically.

## Local Sync Readiness

The sync-readiness feature creates a local export package. It does not send data anywhere.

The exported package includes:

- saved records
- duplicate review output
- local sync queue
- numbering settings
- organization presets
- attendee directory
- custom templates

This gives the project a clean boundary for a future Firebase, Supabase, CRM, Drive, or Methodz API adapter.
