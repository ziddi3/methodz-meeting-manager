# Methodz Meeting Manager v1.6.3 Changelog

## Added

- Portable `provider-contract.js` hosted-provider contract core.
- Structured `ProviderError` codes with explicit retryability metadata.
- Deterministic canonical-JSON conflict tokens.
- Idempotency-key replay and misuse protection.
- Promise-based list, get, upsert, archive, restore, permanent-delete, export, and health operations.
- Disposable `InMemoryHostedProvider` reference implementation.
- Storage-compatible `LocalStorageHostedProvider` reference implementation.
- Reusable `provider-conformance.js` suite.
- Isolated Node conformance coverage for both reference providers.
- Private-JWK and private-key-field rejection from provider exports.
- Hosted-provider contract, architecture, and test documentation.

## Changed

- App-shell and service-worker cache version are now `1.6.3`.
- Meeting-record schema remains `1.6.0`; no migration is introduced.
- The main entry point exposes hosted-provider core and reference adapter browser globals.
- CI runs provider conformance independently from the browser matrix.
- README and Copilot guidance distinguish client contract compatibility from production authentication, authorization, tenant isolation, encryption, durable audit, and legal compliance.

## Preserved

- Browser localStorage remains the active application provider.
- Direct-file operation remains supported.
- No backend, endpoint, token, runtime framework, package installation, or build command is required.
- Existing archive, revision, recovery, retention, preservation, redaction, approval, receipt, signing, verification, and custody controls remain unchanged.

## Security notes

- Provider exports preserve active records, Archive Vault records, revision history, unknown fields, and source integrity metadata.
- Provider exports fail closed when private EC JWK `d` parameters or recognized private-key fields are detected.
- Conflict tokens and FNV-1a package integrity are compatibility controls, not digital signatures or identity proof.
- Passing the conformance suite does not certify a hosted provider’s server-side security, operational resilience, data residency, or legal compliance.
