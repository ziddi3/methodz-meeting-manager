# v1.6.2 Key Custody Test Plan

## Automated Node checks

Run:

```bash
node tests/v162-custody-core.mjs
```

The test must prove:

- public key IDs are derived from normalized P-256 public JWKs
- a completed rotation manifest validates
- no private JWK parameter is present
- tampered key IDs fail validation
- injected private key material fails validation
- same-key rotations fail validation

## Automated browser checks

Run:

```bash
npx playwright test tests/v162-custody.spec.js
```

Coverage includes:

- app-shell version 1.6.2 with record schema still 1.6.0
- visible custody workspace
- completed rotation event storage
- public manifest construction and validation
- private-key exclusion
- fail-closed completed-event checklist behavior

## Manual release gate

1. Open `meeting.html` directly from the filesystem and confirm normal meeting workflows still operate.
2. Serve the repository through localhost and confirm the custody panel loads.
3. Generate two disposable keys.
4. Export each public key and confirm no `d` property exists.
5. Record a planned rotation.
6. Record a completed rotation with all checklist items checked.
7. Export the custody manifest and inspect its public keys and events.
8. Attempt a completed event without a witness. Confirm it is blocked.
9. Attempt a rotation with the same source and replacement key. Confirm it is blocked.
10. Revoke a public key using the v1.6 registry control, refresh custody keys, and confirm the revoked status appears.
11. Clear browser storage and confirm no real private material was ever persisted.
12. Test service-worker offline reload after the new assets have been cached.

## Cross-browser target

The custody core uses standard Web Crypto behavior inherited from v1.6. Browser verification should be exercised in Chromium, Firefox, and WebKit where GitHub-hosted runner availability and Playwright support permit. Failures must remain isolated by browser and upload their own artifacts.
