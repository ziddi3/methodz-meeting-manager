# Methodz Meeting Manager v1.2 Release Notes

## Release theme

Version 1.2 adds explicit approval gates around the two highest-risk local workflows:

1. downloading an external redacted copy; and
2. permanently deleting an archived record.

The app remains static, offline-first, dependency-free at runtime, and directly deployable from `meeting.html`.

## External export approval

External export approval is scoped to all of the following:

- the selected source record or current form;
- a normalized record-content fingerprint;
- the selected redaction profile;
- the selected recipient policy;
- the approval status and optional expiration date.

Changing the source content invalidates the approval. Changing the recipient policy or redaction profile requires a separate approval.

Default recipient policies are:

- Approved Partner Organization: Partner Safe only;
- Public Communications: Public Summary only;
- Contractual / Custom Recipient: Partner Safe or reviewed Custom External Copy.

Only configured approval roles can approve, reject, or revoke an export. The default approval roles are Administrator and Auditor.

Approved JSON and HTML external packages include an approval snapshot and recomputed package integrity. The controlled source record is not modified.

## Permanent disposition approval

Archive Vault permanent deletion now requires:

- no active preservation hold;
- a review request;
- approval by a configured disposition role;
- a matching archived-record fingerprint;
- final confirmation in the existing Archive Vault workflow.

A completed permanent deletion consumes the approval and writes a browser-local audit event.

A preservation hold remains the stronger rule. An active hold blocks disposition even when an earlier approval exists.

## Audit data

Version 1.2 adds browser-local registries and event logs for:

- external export approvals;
- approved external downloads;
- disposition review requests;
- disposition approvals, rejections, and revocations;
- completed permanent deletion events.

Each approval panel can export its approval registry and event history as JSON.

## Security boundary

These controls are workflow safeguards in a local browser interface. They do not provide authenticated identity, immutable logging, server authorization, or non-repudiation.

A future hosted provider must independently enforce:

- authenticated roles;
- approval permissions;
- preservation holds;
- record fingerprint validation;
- disposition authorization;
- durable and append-only audit logging.
