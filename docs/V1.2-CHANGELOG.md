# Methodz Meeting Manager v1.2 Changelog

## Added

- `config-v12.js` for approval roles, recipient policies, and approval storage keys.
- `migrations-v12.js` for approval-governance schema defaults and validation.
- `features-v12-approvals.js` for external-export approval, recipient-policy enforcement, disposition approval, fingerprint validation, and audit export.
- `features-v12.css` for approval and disposition states.
- `archive-v12.js` for read-only approval governance details on archive pages.
- `tests/v12-smoke.spec.js` for approval and fingerprint regression coverage.
- v1.2 release, architecture, and test documentation.

## Changed

- Bumped schema and app-shell version to `1.2.0`.
- External JSON and HTML downloads now require valid approval for the exact source fingerprint, recipient policy, and redaction profile.
- External package integrity is recomputed after approval metadata is attached.
- Archive Vault permanent deletion now requires disposition approval in addition to the existing preservation-hold protection.
- Service-worker cache and CI wiring checks now include v1.2 assets.
- v1.0 and v1.1 smoke tests now validate the current v1.2 schema while retaining earlier feature regression coverage.

## Notes

- Previewing a redacted external copy remains possible before approval; downloading is blocked.
- Browser-local approvals are workflow controls, not authenticated or immutable authorization.
- FNV-1a record fingerprints detect local source changes but are not digital signatures.
- Direct-file operation and dependency-free static deployment remain supported.
