# Methodz Meeting Manager v0.9 Release Notes

## Release Theme

Version 0.9.0 is the migration, archive operations, workspace merge, browser automation, and optional PWA build.

The application remains a static HTML/CSS/JavaScript project. It still works by opening `meeting.html` directly. Hosted deployments gain an optional service worker and installable app shell without changing the core local-record workflow.

## Added

### Stable schema migration registry

`migrations.js` now provides:

- ordered schema migrations
- semantic version comparison
- idempotent record migration
- active-record migration
- archived-record migration
- revision-snapshot migration
- draft migration
- record validation
- migration run metadata under `methodzMigrationState`

The migration registry runs before the main workspace initializes because it is loaded immediately after `config.js`.

### Archive search, filters, selection, and bulk export

`features-v09-archive.js` adds:

- full-text archive search
- meeting-status filtering
- organization / representative filtering
- filtered-record selection
- selected-record JSON export
- filtered-result JSON export
- visible and selected record counts

Archive lifecycle remains non-destructive. Bulk export never removes or rewrites archived records.

### Revision comparison

`features-v09-revisions.js` adds:

- revision-to-revision comparison
- revision-to-current-record comparison
- added, removed, and changed field-path classification
- comparison summary metrics
- readable comparison table
- complete comparison JSON export

Arrays are compared by position. This keeps the implementation deterministic and dependency-free.

### Workspace package merge mode

`features-v09-workspace-merge.js` adds a separate non-destructive import path alongside the existing full replacement restore workflow.

Conflict strategies:

- Prefer the newest record
- Keep local values
- Keep both record versions

The merge workflow:

1. validates the package type and checksum
2. previews incoming record and archive counts
3. saves a complete pre-merge recovery package
4. merges recognized Methodz storage entries by data type
5. writes a merge report to `methodzWorkspaceMergeLog`
6. reloads the application

Complete restore remains available when an exact workspace replacement is required.

### Optional PWA shell

Hosted deployments now include:

- `manifest.webmanifest`
- `service-worker.js`
- `features-v09-pwa.js`
- an SVG application icon
- install prompt handling
- service-worker status reporting
- manual offline-cache refresh

Direct-file compatibility is preserved. Service-worker registration is skipped when the app is opened through the `file:` protocol.

### Automated browser smoke tests

GitHub Actions now runs Playwright smoke tests against a temporary local static server.

The tests cover:

- creating and saving a record
- schema 0.9 record output
- migration of an older record
- non-destructive archive behavior
- archive filter availability
- deterministic workspace merge helpers
- manifest and service-worker availability

Runtime application code remains dependency-free. Playwright is installed only inside the CI job.

## Compatibility Notes

- Existing active records, archived records, revision snapshots, and drafts are migrated in place.
- Migration functions preserve unknown fields.
- Workspace replacement restore remains available in v0.8 form.
- Workspace merge is additive and does not clear local-only storage entries.
- PWA installation requires a supported browser and HTTPS or localhost.
- Opening `meeting.html` directly still supports the complete meeting-record workflow without service-worker caching.

## Version 1.0 Preparation

The v0.9 architecture closes the major pre-1.0 gaps around migration, archive operations, comparison, transfer, and automated browser validation.

The remaining v1.0 work should focus on:

- role-aware record policies
- improved signature consent and verification controls
- asynchronous data-provider compatibility
- attachment storage adapter boundary
- a consolidated full archive workspace
- final schema validation and release hardening
