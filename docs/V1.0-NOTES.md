# Methodz Meeting Manager v1.0 Release Notes

## Release Theme

Version 1.0 turns the accumulated offline feature layers into a release-governed meeting-record system while preserving the original static deployment promise.

The app still runs from `meeting.html` with no build process and no required network connection.

## Added

### Consolidated Records Workspace

A single workspace now indexes active and archived records. It supports:

- full-text search
- active versus archived filtering
- classification filtering
- release-readiness filtering
- active-record editing
- archived-record detail preview
- filtered JSON index export

### Role-Aware Records

Records now store:

- classification
- policy identifier
- allowed roles
- prepared-by and reviewed-by fields
- review status
- protected areas
- policy note

The current workspace role is stored locally and used to gate edit, export, and signature-verification actions.

Because there is no authentication in the static app, these are workflow controls, not a secure identity boundary.

### Signature Consent and Verification

Every attendee row now includes:

- explicit typed-signature consent
- versioned consent statement
- consent timestamp
- verification status
- verifier name
- verification timestamp
- verification note

Saving is blocked when a typed signature exists without recorded consent.

The exact-name-match helper performs text comparison only. It does not claim identity verification.

### Asynchronous Meeting Provider Contract

`async-data-adapter.js` introduces a Promise-based provider interface for future remote storage.

The initial provider wraps the local synchronous adapter. It does not send data anywhere.

### Attachment Provider Contract

`attachment-adapter.js` formalizes attachment-reference storage and validation.

The default provider remains metadata-only and rejects embedded base64 or `data:` file content.

### Final Schema Extension

`migrations-v10.js` adds v1.0 defaults for:

- access control
- signature consent
- signature verification
- attachment adapter metadata
- schema audit
- release metadata

Older signatures are not silently treated as consented.

### Release Gate

The release audit checks active and archived records for schema, governance, consent, task, and attachment issues. Results are stored locally and can be exported.

## Compatibility

- Existing v0.1 through v0.9 records remain migratable.
- Unknown fields are preserved.
- Direct-file mode remains supported.
- The service worker caches application assets only.
- Meeting data remains in local storage unless explicitly exported.
