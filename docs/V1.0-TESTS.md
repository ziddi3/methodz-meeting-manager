# v1.0 Test Checklist

## Direct-File Startup

- Open `meeting.html` directly.
- Confirm the page loads without a server.
- Confirm Record Roles & Policy appears.
- Confirm Signature Consent Review appears.
- Confirm Consolidated Records Workspace appears.
- Confirm Provider Boundaries and v1.0 Release Gate appear.

## Governance

- Change Current Workspace Role.
- Reload and confirm the role persists.
- Create a Restricted Record.
- Remove the current role from Allowed Roles.
- Confirm access evaluation changes.
- Save as Administrator.
- Switch to Participant and confirm restricted edit/export is blocked.
- Switch back to Administrator and confirm access returns.

## Signature Consent

- Enter attendee name and typed signature without selecting consent.
- Save and confirm saving is blocked.
- Select consent and save.
- Reopen the record and confirm consent version and timestamp remain.
- Use Mark Exact Name Matches with identical attendee and signature text.
- Confirm status becomes Name Match.
- Change the signature text so it no longer matches and confirm the helper does not mark it.
- Set verification to Declined and confirm saving a signed row is blocked.

## Migration

- Seed a v0.9 record with a typed signature.
- Reload.
- Confirm the record migrates to schema 1.0.0.
- Confirm the old signature is not automatically marked consented.
- Confirm release validation reports the consent issue.

## Consolidated Workspace

- Save two active records.
- Archive one record.
- Confirm both appear in the consolidated workspace.
- Filter Active Only and Archived Only.
- Filter by classification.
- Filter by release readiness.
- Open an active record.
- Open an archived preview.
- Export the filtered index and inspect the JSON.

## Provider Boundaries

- Run health checks.
- Confirm synchronous, asynchronous, and attachment providers report Ready.
- Export the provider contract template.
- Confirm no network endpoint is present.
- Confirm the async adapter reports `remoteSync: false`.

## Attachment Safety

- Add a normal external reference.
- Confirm it saves and receives adapter metadata.
- Import or construct a reference whose location starts with `data:`.
- Confirm the release audit reports an error.
- Confirm file bytes are not stored in the record.

## Release Gate

- Run the release audit with no records.
- Save a complete record and confirm it is release ready.
- Create an unassigned task and confirm a warning.
- Export audit JSON.

## Hosted PWA

- Serve the repository on localhost or HTTPS.
- Confirm the service worker cache name is `methodz-meeting-manager-v1.0.0`.
- Refresh the cache from the PWA panel.
- Go offline and reload.
- Confirm the v1.0 modules still load.
- Confirm records remain in local storage and are not stored in Cache Storage.
