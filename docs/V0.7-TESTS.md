# Version 0.7 Test Checklist

Run these checks in a modern Chromium or Firefox browser.

For the most consistent storage behavior, serve the repository through a local static server instead of mixing different `file://` locations.

## Startup

- [ ] Open `meeting.html`.
- [ ] Confirm the existing v0.3 through v0.6 panels still load.
- [ ] Confirm the Organization / Representative Directory panel appears.
- [ ] Confirm the Data Adapter panel reports `local-storage` as ready.
- [ ] Confirm no JavaScript errors appear in the browser console.

## Adapter

- [ ] Run **Health Check**.
- [ ] Confirm the displayed record count matches Saved Meeting Records.
- [ ] Save a new meeting and confirm the adapter record count increases.
- [ ] Edit the same meeting and confirm no duplicate record is created.
- [ ] Delete a record and confirm the adapter record count decreases.
- [ ] Export an Adapter Snapshot and inspect the JSON envelope.

## Organization Directory

- [ ] Add a custom organization with a type and primary representative.
- [ ] Confirm the new name appears in the organization checklist.
- [ ] Use **Select** and confirm its checkbox becomes selected.
- [ ] Use **Select + Add Representative** and confirm an attendee row is added.
- [ ] Save the meeting.
- [ ] Inspect the saved JSON and confirm `organizationDetails` contains a snapshot.
- [ ] Edit the directory entry and confirm the older saved record keeps its prior snapshot.
- [ ] Export the directory.
- [ ] Delete the local directory entry.
- [ ] Import the exported directory and confirm the entry returns.

## Archive Page

- [ ] Save a meeting containing attendees, signatures, agenda items, decisions, tasks, attachments, and summary.
- [ ] Choose **Archive Page** on the saved record.
- [ ] Confirm every populated section renders.
- [ ] Confirm missing optional fields use readable empty states.
- [ ] Download the archive record as JSON.
- [ ] Choose **Edit Record**.
- [ ] Confirm `meeting.html` reopens the same saved record for editing.

## Current Meeting Preview

- [ ] Enter unsaved meeting information.
- [ ] Choose **Open Archive Preview**.
- [ ] Confirm the preview renders without creating a saved record.
- [ ] Confirm **Edit Record** is hidden in preview mode.
- [ ] Return to `meeting.html`.

## Print

- [ ] Open a saved archive page.
- [ ] Open browser Print Preview.
- [ ] Confirm action buttons are hidden.
- [ ] Confirm attendance and signature audit are visible.
- [ ] Confirm task and attachment tables are visible.
- [ ] Confirm cards and rows avoid awkward page breaks where possible.
- [ ] Confirm record audit metadata prints near the end.

## Regression

- [ ] Draft auto-save and restore still work.
- [ ] Meeting numbering still advances once for a new saved record.
- [ ] Organization presets still apply.
- [ ] Structured decisions still save and restore.
- [ ] Attachment references still save and restore.
- [ ] Attendee Directory still works and does not save signatures.
- [ ] Duplicate Review still refreshes.
- [ ] Sync queue still receives save and delete changes.
- [ ] Import Preview still stages records before merge.
- [ ] TXT, JSON, HTML, CSV, and PDF workflows still work.

## Failure Tests

- [ ] Open `archive.html?id=missing-record`.
- [ ] Confirm a readable error appears.
- [ ] Import invalid organization directory JSON.
- [ ] Confirm the app rejects it without changing saved entries.
- [ ] Block local storage in browser settings.
- [ ] Confirm the adapter health check reports an error instead of crashing the page.
