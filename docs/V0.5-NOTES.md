# Version 0.5 Notes

Version 0.5 moves Methodz Meeting Manager closer to a real field-ready meeting archive by adding two missing pieces: repeat-attendee handling and evidence tracking.

The app remains static, offline-first, dependency-free, and deployable as plain files.

## New Modules

```text
features-v05-attachments.js
features-v05-directory.js
features-v05.css
```

## Attachment References

Attachment References are structured pointers to supporting material. They are not file uploads.

Each reference can capture:

- reference name
- reference type
- date
- file / evidence location
- added by
- notes

Use this for install photos, quotes, invoices, drawings, logo files, customer communication, CRM references, or other supporting records.

## Attachment Index

Saved meeting records now feed an Attachment Index dashboard.

The index shows:

- total references
- unique reference types
- references missing locations
- meetings containing attachment references

The index can export CSV for review or future migration.

## Attendee Directory

The Attendee Directory stores reusable attendee presets in browser local storage.

Directory entries store:

- name
- organization / role
- attendance type
- notes

Directory entries do not store signatures. Signatures stay meeting-specific.

## Signature Controls

Version 0.5 adds tools to:

- fill unsigned signature fields from attendee names
- remove blank attendee rows
- generate signature audit metadata

The signature helper should only be used when typed-name sign-off is intended and agreed.

## Record Schema Additions

Meeting records can now include:

```json
{
  "attachments": [],
  "attachmentSummary": {},
  "signatureAudit": {},
  "directorySnapshot": []
}
```

## Export Additions

- TXT exports include Attachment References and Signature Audit.
- HTML minutes include Attachment References and Signature Audit.
- Current meeting attachments can export as CSV.
- Saved attachment index can export as CSV.
- Attendee Directory can export as JSON or CSV.

## Design Decision

Binary files are not stored in localStorage. Browser localStorage is not a safe or practical place for large photos, invoices, PDFs, or other binary evidence.

The correct next step is a storage adapter boundary that can later connect to Drive, Firebase, Supabase, CRM storage, or a Methodz API.
