# Version 0.9 Test Checklist

## Automated Checks

GitHub Actions should complete both jobs:

```text
validate-static-app
browser-smoke-tests
```

The browser smoke job validates:

- main workspace load
- save flow
- schema 0.9 output
- older-record migration
- migration state metadata
- non-destructive archiving
- archive filters
- merge helper behavior
- manifest delivery
- service-worker delivery

## Manual Migration Tests

1. Export a complete v0.8 workspace backup before testing.
2. Open browser developer tools and inspect `methodzMeetingRecords`.
3. Change a test record schema version to `0.2.0` and remove optional arrays such as `attachments` and `organizationDetails`.
4. Reload `meeting.html`.
5. Confirm the record still appears.
6. Confirm its schema version is `0.9.0`.
7. Confirm optional arrays were restored without removing unknown fields.
8. Inspect `methodzMigrationState` and confirm the run completed without errors.
9. Reload again and confirm no duplicate records or repeated destructive changes occur.

Repeat with:

- one archived record
- one revision snapshot
- one auto-saved draft

## Manual Archive Tests

1. Save at least three meetings with different statuses and organizations.
2. Archive all three.
3. Search by a title fragment.
4. Filter by meeting status.
5. Filter by organization / representative.
6. Combine search and filters.
7. Confirm hidden records remain stored in `methodzArchivedMeetingRecords`.
8. Choose **Select Filtered**.
9. Confirm the selected count is correct.
10. Export selected JSON.
11. Export filtered JSON.
12. Open both files and confirm only the expected archive entries are present.
13. Restore one archived record and confirm archive controls refresh.
14. Permanently delete a disposable archived record and confirm the selection count refreshes.

## Manual Revision Comparison Tests

1. Save a meeting.
2. Change the title, one attendee, one task, and the summary.
3. Save again.
4. Open **Revision History**.
5. Compare Revision 1 with Revision 2.
6. Confirm changed paths appear.
7. Compare Revision 1 with the current active record.
8. Confirm added, removed, and changed metrics are plausible.
9. Export comparison JSON.
10. Confirm the exported package contains left/right metadata and the complete change list.
11. Compare the same version against itself and confirm no differences are reported.
12. Restore a revision and confirm the comparison controls still work after reopening history.

## Manual Workspace Merge Tests

Prepare two browser profiles or two workspace backup files.

### Prefer newest

1. Create the same record ID in both workspaces with different `updatedAt` values.
2. Choose **Prefer the newest record**.
3. Apply the merge.
4. Confirm the newer record is retained.

### Keep local

1. Create a conflicting record in the local workspace.
2. Choose **Keep local values**.
3. Apply the merge.
4. Confirm the local record remains unchanged.
5. Confirm incoming records with new IDs were added.

### Keep both

1. Create a conflicting record ID with different content.
2. Choose **Keep both record versions**.
3. Apply the merge.
4. Confirm the local record remains.
5. Confirm the incoming copy has a new record ID and an `(Imported Copy)` title suffix.

For every strategy:

- confirm a pre-merge recovery package is stored under `methodzPreRestoreBackup`
- confirm revision maps are deduplicated
- confirm archive entries are merged
- confirm local-only Methodz keys remain
- confirm `methodzWorkspaceMergeLog` records the strategy and source information

## Manual PWA Tests

### Direct-file mode

1. Open `meeting.html` directly from the file system.
2. Confirm the app loads normally.
3. Confirm the PWA panel reports direct-file mode.
4. Confirm service-worker cache refresh is disabled.
5. Save and reopen a record to confirm core functionality is unaffected.

### Localhost or HTTPS mode

1. Serve the repository with a local static server.
2. Open `meeting.html`.
3. Confirm the service worker registers without console errors.
4. Reload once.
5. Disconnect the network or enable browser offline mode.
6. Reload the app.
7. Confirm the static workspace opens.
8. Reconnect and choose **Refresh Offline Cache**.
9. Confirm the browser reports the app as installable when supported.
10. Install the app and confirm it opens `meeting.html` in standalone mode.

## Regression Tests

Run the full `docs/MANUAL-TEST-CHECKLIST.md` after v0.9 checks.

Pay special attention to:

- save and edit
- draft restore
- templates
- structured decisions
- attachments
- attendee and organization directories
- Archive Page navigation
- revision restore
- Archive Vault restore
- complete workspace replacement restore
- adapter contract tests
- keyboard shortcuts
- print output

## Release Gate

Version 0.9 is ready for release when:

- static checks pass
- browser smoke tests pass
- migration is idempotent
- archive exports contain only intended records
- revision comparison does not modify records
- all merge strategies preserve a recovery path
- direct-file mode remains functional
- hosted offline-shell mode works after first successful load
