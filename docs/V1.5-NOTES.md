# Methodz Meeting Manager v1.5.0 Release Notes

## Release theme

Version 1.5 adds local policy operations and external release receipts while preserving the static, offline-first deployment model.

## Added

- Policy stewardship metadata for recipient policies.
- Business-purpose documentation and information-risk tiers.
- Configurable review cadence.
- Review queue states for current, due-soon, overdue, no-date, and inactive policies.
- A review action that advances the recipient policy review date.
- Governance-version binding in redacted previews and approval fingerprints.
- A release receipt after every approved external JSON or HTML download.
- A chained local receipt ledger with verification and JSON export.
- Source-record pointers to the latest release receipt and package digest.
- Policy operations and receipt audit events.

## Deployment contract

The application remains plain HTML, CSS, and JavaScript. No package installation, build command, server, account, or network connection is required for core operation.

The v1.5 module loads after v1.4 so it can wrap the stable recipient-policy and approved-download functions.

## Local data boundary

Policy stewardship, reviews, approvals, and release receipts are stored in browser storage. They support internal workflow but are not a substitute for a hosted identity and authorization service.

A future hosted provider should enforce roles, policy administration, release authorization, and durable audit retention on the server.
