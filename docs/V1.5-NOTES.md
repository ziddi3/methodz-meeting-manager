# Methodz Meeting Manager v1.5 Notes

## Release focus

Version 1.5 adds optional public-key signatures for exported JSON packages while preserving the static, offline-first application model.

## Added

- ECDSA P-256 signatures with SHA-256 through Web Crypto.
- Explicit key generation and JWK import/export.
- Memory-only private-key handling.
- Browser-local public-key registry with Active and Revoked states.
- Signed-package verification and verification-report export.
- Standalone `verify.html` entry point.
- Package-signing and verification audit export.
- Record metadata for the last package-signing operation.

## Key-management rule

The application never writes private JWK material to localStorage. A generated or imported private key exists only in the current page memory. The operator must explicitly download and protect a private-key backup before closing or refreshing the page.

Public keys may be stored locally for repeat verification. Registry status is workflow metadata only. A remote authenticated provider is still required for organization-enforced key administration and durable revocation.

## What a valid signature means

A valid signature confirms that the JSON package content matches the content signed by the holder of the corresponding private key.

It does not independently prove:

- the legal identity of the signer;
- the authority of the signer;
- recipient identity;
- package delivery;
- approval legitimacy;
- compliance with a legal electronic-signature regime.

Verify the public key ID through an independent trusted channel before relying on the signer label.

## Deployment

No runtime dependency, framework, server, or build step was added. Core meeting workflows still operate in direct-file mode. Cryptographic operations require Web Crypto, which is normally available on HTTPS or localhost and may vary in direct-file contexts.
